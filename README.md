# 🏛️ Museum Memory (MVP v1.0)

> **“记录你在博物馆里，那些只属于你的瞬间。”**
- **一个为热爱文化历史、念旧且有规划的旅行者打造的「私人博物馆记忆库」。**
> 🔗 **[立即在线体验 Demo](https://museum-memory-demo.vercel.app)** 

---

## 🌟 项目愿景
本项目旨在解决旅行者照片散落在相册、缺乏深度感悟记录空间的痛点。通过整合「照片 + 文字 + 相关资料链接」，并利用交互式地图展示，唤起用户往日的参观回忆。

- **视觉风格**：干净、温暖、怀旧
- **气质设定**：米白背景 (#FAF7F2)、牛皮纸感、衬线字体
- **核心价值**：私人化记录、零成本维护、极简交互

---

## 🛠️ 技术栈选型
基于 $0 成本快速上线的原则，本项目采用 **AI 辅助开发 (Lovable)** 模式构建。

| 维度 | 技术方案 | 理由 |
| :--- | :--- | :--- |
| **开发平台** | **Lovable.dev** | 自然语言驱动，全栈生成 React 代码，集成度高 |
| **前端框架** | **React + Tailwind** | 响应式支持优秀，UI 组件视觉高级 |

---

## 📊 数据架构
系统核心由三张关联表组成，确保数据的持久化与逻辑清晰：

1. **`museums` (博物馆)**: 存储名称、城市、国家、经纬度坐标及主题标签。
2. **`memories` (记忆条目)**: 关联博物馆，存储参观日期、长篇感悟及参考资料链接。
3. **`photos` (照片)**: 关联具体的记忆条目，存储在 Supabase Storage 的路径。

---

## ✨ 核心功能规划 (P0 Roadmap)
- [ ] **交互式地图**：全屏地图展示，通过城市标记点聚合博物馆，点击跳转详情。
- [ ] **私人记忆页面**：大图网格展示照片，衬线体排版文字感悟，支持外链跳转。
- [ ] **多媒体上传**：支持从设备上传照片至云端存储，自动管理公开 URL。
- [ ] **内容编辑系统**：支持文字感悟的撰写与 2 秒自动保存逻辑。
- [ ] **主题标签系统**：预设“历史、艺术、考古”等标签，支持自定义分类。
